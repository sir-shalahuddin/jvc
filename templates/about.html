Behind the Scenes

How we built a performant AI-powered retrospective tool on GCP Always Free Tier.

🚀 How it Works

1. Session Orchestration

Users create a secure retrospective session. Each session is a unique document in Firestore, managing real-time state for participants.

2. Anonymous Reflections

Participants submit feedback anonymously. To ensure high quality, we use Quill.js for rich text and bluemonday for server-side HTML sanitization.

3. AI Sentiment Analysis

The core intelligence is powered by Google Gemini AI. As feedback is submitted, our custom background worker leverages Gemini to perform deep sentiment analysis and contextual emotion mapping.

4. Safety & Guardrails

Utilizing Gemini's safety filters, the UI automatically highlights toxic content with a pulsing red alert for real-time moderation.

🛠️ Backend Architecture

Golang Google Cloud Run Google Firestore Google Gemini AI Clean Architecture

The system is built with a Clean Architecture approach, ensuring the application is easy to maintain, test, and scale on Google Cloud's serverless infrastructure.

⚡ High Availability & RPM Handling

To overcome Gemini API rate limits and ensure 100% uptime, we implemented a sophisticated failover system:

  • Batch Processing: Multiple feedback items are bundled into a single Gemini API request, reducing RPM usage by up to 90%.
  • Model Failover: If a rate limit is hit, the system automatically switches between a pool of secondary models in real-time.
  • Worker Concurrency: Go routines manage background analysis without blocking the main user experience.
